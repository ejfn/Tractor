## Review Findings & Fixes Summary

### Module 1: `src/types/`
- **Deserialization Type Safety**: Replaced unsafe casting in `deserializeCard` with a robust `isCardLike` type guard.
- **AI Game Context Tightening**: Hardened `trumpInfo` to be required in `AIGameContext` rather than optional.
- **Enum & Suit Exclusions**: Constrained `suitVoids` to `Set<PlayableSuit>` (filtering out `Suit.None`) and updated distribution definitions.
- **Callers Refactoring**: Updated all game state initializers and AI managers to conform to the stricter types.

### Module 2: `src/utils/`
- **Card Sorting Type Hardening**: Refactored `getSuitOrderValue` and `getRankOrderValue` to accept exact `Suit`/`Rank` enums rather than broad strings.
- **Joker Suit Logic**: Removed redundant joker ternary branches in `suitHelpers.ts` where both paths returned the identical value.
- **Translation Utility Safety**: Tightened the typing of translation handlers and parameters, replacing generic `any` with strict typing.
- **Fisher-Yates Shuffle Implementation**: Replaced biased `sort(() => 0.5 - Math.random())` in `gameInitialization.ts` with a proper Fisher-Yates array shuffle.
- **Persistence Infinite Retry Bug**: Discovered and resolved a bug in `gameStatePersistence.ts` where calling `getItem` in a retry block passed the static `retries` parameter instead of incrementing it, leading to a possible infinite retry loop.

### Module 3: `src/game/`
- **Card Value Records**: Removed redundant nullish coalescing `|| 0` on exhaustive `Record<Rank, number>` properties.
- **Tractor Logic Type Safety**: Cleaned up suit offsets in `tractorLogic.ts`, transitioning from string lookups to enum values.
- **Multi-Combo Grouping**: Cleaned up unreachable fallback branches in suit grouping functions.
- **Combo Detection Deduplication**: Extracted a duplicated `getLeadingSuit` helper from `playValidation` and `comboDetection` into a single, clean module-level exported function in `comboDetection.ts`.
- **Validation Rules Alignments**: Adapted the rules engine to import unified helpers rather than recreating local equivalents.

### Module 6: `src/ai/following/`
- **Dead Filter Body [BUG]**: `attemptToBeatWithThreshold` in `validCombosDecision.ts` used a block-body arrow `(combo) => { combo.cards.some(...); }` — the `.some()` result was always discarded, so `strongBeatingCombos` was always empty and the threshold-gated beat path was completely unreachable. Fixed with an expression-body arrow.
- **In-Place Hand Mutation [BUG]**: `selectCrossSuitDisposal` in `multiComboFollowingStrategy.ts` called `playerHand.sort()` directly, silently reordering the caller's actual hand. Fixed with `[...playerHand].sort()`.
- **Unsafe Type Assertions [TYPE]**: Two `as PlayerId` casts in `teammateAnalysis.ts` on `gameState.players[idx]?.id` (`string | undefined`). Since `playerMemories` is `Record<string, PlayerMemory>`, no cast is needed at declaration or lookup. Both removed; the existing `&& nextPlayerId` guards handle narrowing correctly.
- **Dead Code Removal [STYLE]**: `fallbackSelection` in `routingLogic.ts` declared `trumpInfo?` as optional but it was always provided. The unreachable `else` branch was removed and the parameter made required.
- **Case Block Scoping [STYLE]**: Added `{ }` blocks around `const` declarations in `case ComboType.Pair` and `case ComboType.Tractor` in `suitAvailabilityAnalysis.ts` to eliminate lint warnings for lexical declarations in case clauses.

### Module 7: `src/ai/leading/`
- **`isTrump` Shadowing [TYPE]**: `createMetadata` in `candidateLeadDetection.ts` declared `const isTrump = suit === Suit.None` which shadowed the imported `isTrump` function for the entire scope. Renamed to `isTrumpSuit` to eliminate the shadowing hazard.
- **`createMemoryContext` in Loop [PERF]**: `getAllUnbeatableCardsInSuit` recreated an identical `MemoryContext` on every loop iteration. Hoisted the call above the loop since `gameState` is constant across iterations.
- **`voidSuits` Type Tightening [TYPE]**: `LeadingContext.teammate.voidSuits` and `opponents.voidSuits` were typed `Set<Suit>` despite never containing `Suit.None` (trump voids are tracked separately via `isTrumpVoid`). Changed to `Set<PlayableSuit>` in the interface and both constructors; added `as PlayableSuit` casts at the one loop add-site and the two `scoreNonTrumpLead` has-call sites where the broader `Suit` type was passed in. Also imported `PlayableSuit` into `leadingScoring.ts`.

### Module 8: `src/ai/kittySwap/` + `src/ai/trumpDeclaration/`
- **Dead `shouldDeclare` Variable [DEAD]**: `getAITrumpDeclarationDecision` in `trumpDeclarationStrategy.ts` set `const shouldDeclare = true` then used it in a ternary `shouldDeclare ? bestSuitEvaluation.declaration : undefined` — the false branch was unreachable. Inlined as `shouldDeclare: true` and `declaration: bestSuitEvaluation.declaration` directly.
- **Redundant `if (card.rank)` Guard [TYPE]**: `calculateSuitStrength` guarded the map update with `if (card.rank)`, but `rank` is a required `Rank` enum field (non-empty string) — always truthy. Guard removed.
- **In-Place `.sort()` on Map Entry [STYLE]**: `evaluateAllPossibleTrumpSuits` called `options.sort(...)` directly on the array stored in `suitGroups` Map, mutating it in-place. Changed to `[...options].sort(...)`.
- **Redundant `pair.cards[0].rank &&` Guard [TYPE]**: `applyBasicExclusions` in `kittySwapStrategy.ts` guarded `bigRanks.includes(pair.cards[0].rank)` with a leading `pair.cards[0].rank &&` — redundant since `rank` is always defined. Guard removed.
- **Redundant `if (card.suit)` Guard [TYPE]**: `selectFromDisposableCards` wrapped the suit-group logic in `if (card.suit)` — `card.suit` is a required `Suit` enum, always truthy. Guard removed.
- **Header Comment Mismatch [STYLE]**: Module header said "No big pairs (rank > 7)" but `bigRanks` starts at `Rank.Five` (i.e. rank ≥ 5). Corrected to "rank >= 5".

### Module 9: `src/ai/llm/`
- **Dead `.replace()` Calls [DEAD]**: `buildLLMSystemPrompt` in `llmGamePrompt.ts` applied 4 regex `.replace()` calls to `STATIC_LLM_GAME_RULES` looking for `${trumpRankStr}`, `${trumpSuitStr}` etc. — none of these placeholders exist in the template string, so all 4 were no-ops. Trump values are injected via `buildUserPromptTemplate` args, not the system prompt. Simplified to `return STATIC_LLM_GAME_RULES` directly.
- **Unused Interface Field [TYPE]**: `handCardsCount: number` was declared in `UserPromptTemplateArgs` in `llmPromptTemplates.ts` and populated at the call site, but never referenced inside `buildUserPromptTemplate` or any block builder. Removed from interface and call site.
- **Redundant `config.applyToPlayers &&` Guard [TYPE]**: `callLLMForDecision` in `llmAIStrategy.ts` guarded `.includes()` with a leading `config.applyToPlayers &&` — `applyToPlayers` is a required `string[]` field initialized in `DEFAULT_LLM_CONFIG`, never undefined. Guard removed.
- **`allCategories` Mutation [STYLE]**: `localBuildHandDisplay` in `llmGamePrompt.ts` spread `categoryOrder` into a mutable array then pushed extras into it. Refactored to a declarative `filter` + spread concat.

### Module 10: `src/locales/`
- **`console.warn` / `console.error` [STYLE]**: `getStoredLanguage` and `changeLanguageCustom` in `index.ts` used raw `console.warn`/`console.error`. Replaced with `gameLogger.warn`/`gameLogger.error` per codebase convention; added `gameLogger` import.
- **Redundant `as Language` Cast [STYLE]**: `getStoredLanguage` cast `stored as Language` after a `=== "en" || === "zh"` condition that already narrowed the type to `Language`. Removed the cast to rely on TypeScript's native narrowing.
